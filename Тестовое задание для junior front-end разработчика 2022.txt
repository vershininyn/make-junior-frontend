Тестовое задание для junior front-end разработчика


Необходимо реализовать адаптивную верстку страницы согласно представленному графическому макету. Последний доступен в Figma и InVision, а также в нашем облаке. Ограничение по времени выполнения задания — 5 рабочих дней.


Ограничений при выборе технологий и инструментов для разработки нет, но не придумывайте ничего нового. Применяйте только те библиотеки и паттерны, которые вы бы использовали на реальном проекте.


Рекомендуемый стек технологий:
— ES6
— CSS Grid Layout + Flexbox Layout
— БЭМ
— webpack/Gulp + ESLint + SassDoc


При написании кода мы приветствуем адекватные комментарии. Плюсом будет использование SassDoc (или альтернатив) для стилей (это желательно, но не обязательно).


Выполненное задание отправьте, пожалуйста, в телеграм @titaeva или на email office@make.st в виде архива. Желательно (но не обязательно) разместить публичный репозиторий на Github/Bitbucket. Укажите какой объем работы был выполнен, и что для этого использовалось.
Список задач


1. Адаптивная верстка всех блоков согласно «сетке» макета. В зависимости от разрешения экрана устройства последняя может быть:


— «Десктопной» двенадцатиколоночной (ширина вьюпорта равна или больше 1200 px), с постоянным зазором в 32 px между колонками переменной длины; ширина контента находится в интервале 1072–1312 px;


— «Планшетной» восьмиколоночной (ширина вьюпорта от 736 до 1200 px), с постоянным зазором в 32 px между колонками переменной длины; ширина контента — 672–928 px;


— «Мобильной» четырехколоночной (ширина вьюпорта менее 736 px), с постоянным зазором в 24 px между колонками переменной длины; ширина контента — 296–424 px;


2. Написание базовых стилей для типографики. Интерлиньяж = кегль шрифта + 8. На телефонах размеры заголовков и текста переопределяются (см. макет). Для всего текста на странице используется шрифт Graphik LCG, который можно взять отсюда;
3. Написание стилей для элементов интерфейса: анимация при наведении курсора на кнопки и переключатели, анимация нажатия, стили для заблокированных (неактивных) элементов. Каждый первый из них должен быть интерактивным; остальные элементы того же типа должны просто отражать фиксированное состояние (нажатая кнопка, открытый выпадающий список, и т.д.);
4. Реализация полей ввода с нестандартным «плейсхолдером», переходящим в текстовую подсказку при фокусе на поле;
5. Реализация «аккордеона», скрывающего/показывающего текстовый контент произвольного объема с плавной анимацией
Критерии оценки


1. Соответствие верстки макету. Мы будем проверять параметры сетки, отступы, поведение разметки при изменении размеров экрана, ее внешний вид на реальных мобильных устройствах;
2. Внимание к деталям. Выровнен ли «треугольник» выпадающего списка по вертикали относительно текста? Правильный ли интерлиньяж задан для заголовков? и т.д.
3. Стиль оформления кода. Консистентность отступов и их типа (либо табуляция, либо пробелы), отсутствие лишних пробелов в конце строк, осмысленные названия переменных, отсутствие «мусора» в виде закомментированных участков кода и отладочных выводов;
4. Разумное использование современных технологий. Реализация сетки на float с clearfix — это хорошо, но желательно использовать более подходящие и современные инструменты с хорошей поддержкой браузеров;
5. Наличие базовой документации к коду. Если используется сборщик вроде Webpack/Gulp/Parcel — желательно привести небольшую инструкцию по установке всех необходимых зависимостей и выполнению сборки ассетов. В коде JavaScript желательно приводить осмысленные комментарии для методов и свойств классов, в идеале — использовать JSDoc (или аналоги).
